File & Folder Structure

This describes the file structure of Itodo React application template.

Path Description
– public This folder contains CSS, JS, Images, and Plugins files of application.
      — assets This contains necessary Assets used for application.
            — css This contains CSS. CSS stands for Cascading Style Sheets. It’s used to define styles for web pages including the design, layout, and variations in display for different devices and screen sizes.
            — fonts This contains fonts including font awesome library used for the application.
            — images This contains images used for the application.
            — js This contains third parties JavaScript and site js files used for the application.
      — handlers This directory contains Handlers for specific operations in application.
            — phpmailer This directory contains necessary files for php eMail operations
      — index.html The index.html lives at the root of the front-end structure.
– src Entire source code of the application is available in this folder so it is the main folder.
      — app Sub-directory of 'src', containing pages/components
            — components All webpages will go here. Each page is a react component and can have child components also.
            — elements This directory contains independent components used anywhere throughout the template.
            — sections This directory contains all child components or sections of main webpages.
      — globals All global data/constants for the tempalate will go here.
            — banner.js The raw JSON data file contains dummy data for inner pages banners.
            — constants.js This js file contains constant variables and functions used in entire template.
            — data.js The raw JSON data file contains dummy data for the tempelate used at different places.
      — layouts The layout directory contains a brief structure or data flow framing, can be used with any component to contain inside or any component can be used with the specific layout as child.
      — routing This directory contains data flow routing or components accessing flow for the template.
      — App.js This is a main application java script  file.
      — Index.js Whenever an application is run, the First Index.js file is a load.
– package.json A package.json file contains metadata about an app or module. It includes the list of dependencies so it’s most important for the application.

 

Installation

This section guides you to install all required software and libraries to utilize itodo react for your projects fully.

Node

Node.js® is a JavaScript run time built on Chrome’s V8 JavaScript engine. Node.js is required to build the Itodo React. Download the latest version of Node and install it: Download Nodejs

NPM

NPM is the package manager for JavaScript and the world’s largest software registry. NPM is a separate project from Node.js and tends to update more frequently. As a result, even if you’ve just downloaded Node.js (and therefore npm), you’ll probably need to update your NPM .

npm install --global npm@latest

Verify that npm in successfully installed, and version of installed npm will appear

npm --version

React

Installation

1. First, unzip the zip file that you have download from ThemeForest. You will find a itodo-React-1.0 project folder.

2. Then after Extract zip File (itodo.zip) into a folder, you have to work within.

3. Open a Node Prompt. Go to your Working Folder, And run the Following Command and wait for finish installation.

npm install

4. This command will install all the required node module into node_module/ inside your working folder.

5. You are ready to run the itodo react application for the first time

6. While still your working folder and run the following command

npm start
Deployment

Before the production, you must be changed in this Package.json file. In this file replace homepage attribute in your server URL.see the below screenshot.

Then after open node command prompt and go to the project directory. Type following command

Now, Build is Completed and build folder is created in your project root directory.

Open after build folder and change in the index.html file.in this link tag in href attribute and script tag in src attribute both in add your server path.

Adding New Component

It’s easy to add your own custom component in this itodo react application if needs. Please follow the below steps for that.

Step 1 :

>> Please go to src->comp->Sections path and create a new file.
>> Please refer below reference code or get a reference from the existing component which is already done in the project. 


    import React, { Component } from 'react';
    class ComponentName extends Component {
        constructor(props) {
          super(props);
        }
        render() {
            return (
                    <div>
                        // Component Code Here...
                    </div>
                 )
        }
    }
    export default ComponentName;

Step 2: Now you can create a function for getting data or calculate data. Please refer below code.


    FunctionName(functionParam){
     
        this.setState({
            ... // Set The Value in Set useing Function
        })
        
        // Function Logic.
     }

Step 3: Function can be called by following ways.

      3.1  If it needs to call function immediately after a component is mount then call the function as per below example.


    componentDidMount() {
        ...
        this.FunctionName();
    }

      3.2 If it is need to call function once the component render then call the function as per below example.


    render() { 
        return( 
          <div> 
            ... 
            {this.FunctionName()}
          </div>
        ) 
      }

      3.3 if it needs to call function with return data then it can be call by following way.


    render()
    {
       return(
           <div>
              ...
              <ComponentName functionreturn={this.FunctionName()} />

           </div>
           )
    }
